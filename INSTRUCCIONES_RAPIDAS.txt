INSTRUCCIONES RÁPIDAS - CONFIGURACIÓN DE URLs
=============================================

PROBLEMA RESUELTO:
- Tu proyecto tenía URLs hardcodeadas con /subs/ que funcionaban en local pero no en producción
- Se ha implementado una solución centralizada

ARCHIVOS CREADOS:
1. config/app.php - Configuración para desarrollo (BASE_URL = '/subs/')
2. config/app.production.php - Configuración para producción (BASE_URL = '/')
3. utils/url_helpers.php - Funciones helper para URLs dinámicas
4. switch_environment.bat - Script para cambiar entre entornos

CÓMO CAMBIAR ENTRE ENTORNOS:

PARA DESARROLLO LOCAL:
- Usa config/app.php (ya configurado con BASE_URL = '/subs/')
- Tu aplicación funcionará en: http://localhost/subs/

PARA PRODUCCIÓN:
- Ejecuta: switch_environment.bat y selecciona opción 2
- O manualmente: renombra config/app.production.php a config/app.php
- Tu aplicación funcionará en: https://sie.hjconsulting.com.ec/

ARCHIVOS MODIFICADOS:
- index.php (ahora usa constantes de configuración)
- controllers/AuthController.php (URLs dinámicas)
- views/auth/login.php (funciones helper)
- views/admin/dashboard.php (funciones helper)
- public/js/login.js (URL base dinámica)

FUNCIONES HELPER DISPONIBLES:
- url($path) - Genera URL completa
- css($file) - URL para archivos CSS
- js($file) - URL para archivos JavaScript
- image($file) - URL para imágenes
- login_url() - URL de login
- logout_url() - URL de logout

EJEMPLO DE USO EN VISTAS:
Antes: <link rel="stylesheet" href="/subs/public/css/styles.css">
Después: <link rel="stylesheet" href="<?php echo css('styles.css'); ?>">

Antes: <a href="/subs/admin/dashboard">Dashboard</a>
Después: <a href="<?php echo url('admin/dashboard'); ?>">Dashboard</a>

COMPLETAR LA MIGRACIÓN:
1. Ejecuta switch_environment.bat para cambiar a producción
2. Prueba la aplicación en ambos entornos
3. Si hay más archivos con URLs hardcodeadas, usa el patrón mostrado arriba

BENEFICIOS:
✓ Un solo archivo para cambiar entre entornos
✓ URLs dinámicas que se adaptan automáticamente
✓ Mantenimiento fácil y centralizado
✓ Sin conflictos en el control de versiones
